# 工具安装与降级策略

本文件用于指导使用者在机器上缺少 Unity 逆向工具时，如何按需安装，而不是被动卡住。

## 总原则
不要把“缺少某工具”直接说成“当前任务无法继续”。

应按以下顺序处理：
1. 先盘点已有工具
2. 先说明已有工具能完成哪一部分
3. 再指出为了继续推进，缺少的最小必要工具是什么
4. 给出按需安装建议
5. 如果用户暂时不想安装，给出降级路径

## 默认先检查的基础工具
- `python3`
- `pip3`
- `strings`
- `xxd`
- `hexdump`
- `dotnet`

如果这些就已经存在，很多盘点/抽样/轻量提取任务都能推进。

## 推荐优先安装顺序
### 1. UnityPy
适合：
- 枚举 Unity 资源对象类型
- 抽样导出图片、音频、文本对象
- 快速盘点 `.assets` / `sharedassets*` / `resources.assets` / bundle

安装方式：
- `python3 -m pip install UnityPy`

如果安装失败：
- 先记录失败原因
- 再退回到 `strings` / `xxd` / `hexdump` + 文件级盘点

### 2. 更强的 .NET / IL 反编译工具
适合：
- 想继续深入 Managed 程序集
- 需要看类、字段、方法逻辑

如果当前机器只有 `dotnet`，但没有成熟反编译工具：
- 应如实说明当前只能做到程序集盘点、字符串层入口发现、metadata 锚点分析
- 不要装作已经完成 IL 级反编译

### 3. 更强的 Unity 专项导出工具
适合：
- UnityPy 无法完整导出音频/图片
- 需要更完整的资源恢复能力

是否安装取决于：
- 用户是否允许
- 当前任务是否真的被工具能力卡住

## 当用户不想安装工具时
要给出降级路径，而不是只说“做不了”。

### 对图片任务
可降级为：
- 先做对象类型盘点
- 先确认是否存在 `Texture2D` / `Sprite`
- 只记录对象名、来源文件、样本信息

### 对音频任务
可降级为：
- 先确认是否存在 `AudioClip`
- 若不能导出，记录对象名、来源容器、失败现象
- 明确下一步需要更强工具

### 对关卡任务
可降级为：
- 先做 `TextAsset` / `.bytes` / metadata / Managed 字符串锚点排查
- 先识别层次：布局层 / 正文层 / 展示层 / 系统层 / 引用桥层

## 推荐表述方式
推荐这样说：
- “当前机器已有 `python3`、`strings`、`dotnet`，先足够做一轮盘点和样本验证。”
- “如果你要继续批量导出图片/音频，建议补装 `UnityPy`。”
- “如果你要继续看 Managed 逻辑，当前还缺更强的 .NET 反编译工具。”
- “如果你现在不想安装工具，我可以先按已有工具把资源分层和可疑入口盘出来。”

避免这样说：
- “没装 X，所以没法分析 Unity。”
- “必须先装完所有工具再开始。”

## 最终目标
工具安装建议必须服务于当前任务，而不是为了把机器装满。

优先做：
- 能缩小不确定性的最小动作
- 能验证样本的最小安装
- 能支撑下一层分析的最小依赖
